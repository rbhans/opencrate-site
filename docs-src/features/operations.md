# Operations Workflows

## Discovery to acceptance

The operations lifecycle starts with discovery:

- scan BACnet or Modbus networks
- inspect discovered devices and points
- accept what belongs in the project
- apply names, templates, and tags

That flow bridges the protocol layer and the project model. It is one of the most important places where raw field data becomes an operator-friendly system.

## Day-to-day operation

Once a project is commissioned, operators mainly live in:

- alarm views
- point and equipment pages
- schedule editors
- history and dashboards
- floor plans and site maps

Those screens are all backed by the same core stores and realtime updates, which keeps the desktop and API-backed experiences aligned.

## Configuration and accountability

OpenCrate treats administrative actions as first-class operational data:

- notification rules
- webhook targets
- user roles
- commissioning status
- sign-off events
- audit trail entries

That matters because BMS changes are rarely isolated. Alarm routing, schedules, device config, and sign-off workflows all need traceability.
