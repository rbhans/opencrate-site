# Core Features

OpenCrate has grown into a feature-rich BMS platform. The easiest way to understand it is by grouping capabilities into operator workflows.

## Device onboarding

- protocol bridges and discovery flows find devices and points
- commissioning tools track verification and sign-off work
- templates and profiles help standardize accepted devices

## Operations

- alarms support active views, history, acknowledgements, shelving, and routing
- schedules support weekly and exception behavior with BACnet schedule sync
- history collection supports interval and change-based trending
- floor plans and site maps provide spatial context for equipment and points

## Automation and analysis

- a visual logic system compiles into executable logic
- reporting turns runtime data into scheduled reports and exports
- energy features track meters, rates, baselines, and rollups
- FDD rules detect common system issues and maintain fault history

## Integrations and delivery

- MQTT publishing distributes event and value updates
- webhook providers deliver external notifications and integrations
- email, SMS, and webhook channels support routed alarm delivery

## User model

- authentication and role-based permissions gate administrative actions
- audit logging records operator actions across major workflows
